# 今後勉強するなら、どのプログラミング言語？
今回はプログラミング言語を本腰を入れて勉強しようという方に対してお勧めしたいと考えた言語を紹介しようと思います。

## 近年話題となっている言語
単純に話題になっている言語という意味でしたらTIOBE Indexで、ある程度定量的に確認することが出来ます。
フォーマルということであればこの指数が一番手かもしれません。

>__TIOBE Index__  
>TIOBE Index（ティーオーブ・インデックス）とは、オランダのTIOBE Software社が毎月更新・公開しているプログラミング言語の人気度ランキングです。 
これは、プログラミング言語の優劣や書かれたコードの行数を示すものではなく、あくまで「その言語が世間でどれだけ話題になっているか、注目されているか」を測るための指標です。 
主な特徴と仕組み
算出方法: Google、Bing、Yahoo!、Wikipedia、Amazon、YouTube、Baiduといった主要な検索エンジンの検索クエリ（検索ワード）数に基づいて人気度を測定しています。
目的: 開発者が「自分のプログラミングスキルが時流に合っているか」を確認したり、新しいプロジェクトで採用する言語を決定したりする際の参考情報として利用することを目的としています。
歴史: 長期間にわたるデータ（2000年以降）を公開しており、言語の人気トレンドを時系列のグラフで追うことができます。
最新動向: 近年はPythonが首位を独走しており、C、C++、Java、C#といった言語がそれに続いています。

Pythonは鉄板ですね。
その他はCやC++、Javaなどが並びます。

## 将来性が高いプログラミング言語とは
但し、プログラミング言語の話題になっているからと言って、将来性があるかというと少し別の話です。
将来性が高いプログラミング言語は、単に「流行している」だけでなく、技術的な優位性、市場の需要、そしてコミュニティの活動という複数の条件を満たしていると考えてます。


### 1. 市場・産業における需要と汎用性

最も重要な条件は、**多くの産業で採用されているか**、または**成長分野の核となっているか**という点です。

* **成長分野への貢献度が高い**
    * AI、データサイエンス、クラウドインフラ、Web3.0など、現在最も成長している分野の**中心的な開発言語**であること（例: **Python**の機械学習、**Go**のクラウド）。
* **汎用性が高い**
    * 特定の用途（例: データ分析のみ）に限定されず、**サーバーサイド、フロントエンド、モバイル、組み込み**など、複数の開発領域で利用できること（例: **JavaScript/TypeScript**、**Python**）。
* **大規模システムでの実績**
    * スタートアップだけでなく、**大手企業や金融機関**など、信頼性や拡張性が求められる大規模システムでの導入実績が豊富であること。


### 2. 技術的な優位性と効率性

言語仕様そのものが、現代のコンピューティング環境や開発手法に適しているかという点も重要です。

* **高速性・効率性**
    * CPUやメモリを効率的に使い、**処理速度が速い**こと（例: **Rust**、**Go**、**C/C++**）。特に大規模な並行処理や低レイテンシ（低遅延）が求められるシステムで有利です。
* **安全性・堅牢性**
    * **メモリ安全性**や**型安全性**に優れ、実行時エラーやセキュリティ上の脆弱性を事前に防ぎやすい設計であること（例: **Rust**、**TypeScript**）。バグの少ない、持続可能なコードを書くのに役立ちます。
* **並行処理・非同期処理への対応**
    * マルチコアCPUを最大限に活用するための**並行処理（コンカレンシー）**や**非同期処理**の仕組みが言語に組み込まれていること（例: **Go**、**Rust**）。


### 3. コミュニティとエコシステムの成熟度

言語の継続的な発展と、開発者の生産性を支える基盤が整っていることも不可欠です。

* **活発なコミュニティ**
    * バグの修正や新しい機能の提案が活発に行われ、**ドキュメントや情報が豊富**であること。
    * **GitHubの活動量**や、Stack Overflowなどでの質問・回答が充実していること。
* **豊富なライブラリとフレームワーク**
    * 開発を効率的に進めるための**豊富なサードパーティライブラリや使いやすいフレームワーク**（生態系、エコシステム）が整備されていること（例: PythonのAI系ライブラリ、JavaScriptのReact/Vue/Angular）。
* **継続的な進化**
    * 言語の仕様が古びることなく、**最新の技術トレンド**に合わせて定期的にアップデートされ、進化を続けていること。

これらの条件を多く満たしている言語は、将来にわたって高い需要が見込まれ、学習する価値が高いと言えます。

## 将来性が高いと感じる言語

著者が、上記の「将来性が高いプログラミング言語の条件」を総合的に満たしており、現在最も将来性が高いと考えているプログラミング言語は、主に以下の3つです。


| 言語 | 満たしている主な条件 | 成長分野 |
| :--- | :--- | :--- |
| **Python** | 市場需要（最大）、成長分野への貢献度（AI/データサイエンス）、コミュニティの成熟度。 | AI/機械学習、データ分析、バックエンド開発 |
| **Rust** | 技術的な優位性（安全性・高速性）、並行処理、クラウドインフラ（Wasm）。 | システムプログラミング、ブロックチェーン、クラウドネイティブ |
| **TypeScript** | 汎用性の高さ（フルスタック）、安全性（型安全性）、市場の需要（Web開発）。 | 大規模フロントエンド、バックエンド（Node.js） |


### 1. Python (パイソン)

Pythonが圧倒的な将来性を持つ最大の理由は、その**市場需要**と**エコシステム**が、現在の技術トレンドの核に位置しているためです。

* **成長分野への貢献度:** **機械学習**と**データサイエンス**においてデファクトスタンダードであり、この分野の成長が続く限り、Pythonの需要は減りません。
* **コミュニティとエコシステム:** TensorFlow、PyTorch、NumPy、Pandasなど、世界中の開発者や研究者が利用する**極めて豊富なライブラリ**群が、開発者の生産性を圧倒的に高めています。
* **汎用性:** Web開発（Django, Flask）やスクリプト処理など、幅広い用途で使われています。

### 2. Rust (ラスト)

Rustは、技術的な優位性の観点から将来性が極めて高いと評価されています。

* **安全性・堅牢性:** **メモリ安全性**を言語仕様レベルで保証しているため、C/C++で頻繁に発生するクラッシュやセキュリティ脆弱性を防げます。これは、GoogleやMicrosoftといった大企業が、従来のシステムコードの一部をRustに置き換え始めている最大の理由です。
* **高速性・並行処理:** C++と同等の実行速度を持ちながら、安全に**ゼロコスト抽象化**を実現し、並行処理を効率的に記述できます。
* **市場への浸透:** クラウド（AWS、Azure）、WebAssembly (Wasm)、ブロックチェーンといった**次世代のインフラストラクチャ**での採用が急速に拡大しています。

### 3. TypeScript (タイプスクリプト)

JavaScriptのエコシステムの巨大さと、TypeScriptによる品質向上という二つの要素で将来性を確保しています。

* **汎用性と需要:** ベースであるJavaScriptがWeb（フロントエンドとバックエンド）の事実上の標準であり続けているため、市場での需要は極めて高いです。
* **安全性・堅牢性:** 静的型付けを導入することで、大規模なJavaScriptプロジェクトにおいて**コードの可読性、保守性、堅牢性**を飛躍的に向上させました。これにより、大規模開発やエンタープライズ分野で急速に採用されています。
* **継続的な進化:** Microsoftが開発を主導しており、JavaScriptの進化（ECMAScript）に追従しながら、継続的に機能が強化されています。



## 結局言いたいことは？
Python + Rust or TypeScriptがベストだと感じてます。

Pythonがよく使われているからと言って、仕事でPythonしか使わないということはありませんでした。
システムのUIや処理はPython、内部の高速処理はCやC++、Rustというように別言語を使うということはよくあります。

使われて、廃れづらい、どんな場面でも使われる、であればPythonでしょう。
Pythonが苦手にしている処理や限られたリソースで実行するを補えるRustか。
もしくは、組込みのような限られたリソースとはほぼ無関係で良い、という場合はWebシステムでマストなJavaScriptか、上位互換のTypeScriptが良いと思います。
